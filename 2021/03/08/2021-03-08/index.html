2021.03.08 协助React面试有感

这次并不是自己被面试，而是协助架构师去面试 Team Leader 的角色，更多情况只是作为一个翻译，协同被面试者和面试官沟通。

大多数面试的点，在React 电面复盘里都提到了，不过因为这次面试的是 TL 的角色，所以又增加了一些关于性能优化检查和多端开发的知识点。

这一点，在如何写简历里，经验1-4年这个分支里面也有提到过：


这次面试的仁兄挂在了这几点上：

  • 软技能 -> 沟通

    英文真的是拖后腿了，很多回答的点面试官无法理解。而且因为沟通无法非常流畅的进行，所以面试官有更多的时间准备下一个面试题。

  • 进阶

    • 性能指标

      这里真的完全没有回答上来，这也提醒我了，之后应该更加注意这方面的细节。毕竟之前更多想的是完成一个功能的实现，还没有达到提升功能性的方面。

    • Native 开发

      这位仁兄的确将 Native 开发作为经验放到了简历上，但是一被问到就露怯了。个人感觉，自己不是很自信的项目就尽量不要放到简历上了，问到了又无法回答出来，这也并不是一个加分项。

  • 基础

    • React

      问到了生命周期函数都无法回答上来，这个感觉基础是真的有点儿不太靠谱。

      知道要准备面试也不临时抱佛脚一下，基本上所有 React 面试都会问到生命周期啊。

    • http

      300 段的 code 问到了，也无法顺利地回答上来。

这位仁兄的优点也是有的：

  • JS 方面的基础还不错，至少面试官给他了一段代码能够回答出正确的答案；

  • 简历上看起来还不错，小程序的开发、Native 的开发都有写到。

2021-03-12 加更：

新帮着面试了一位候选人，基础方面都没什么问题，可惜用的技术好像是 Angular 而不是 React。这种情况下，不知道有没有大佬能提出给被面试者的建议？

如果有这种机会，是应该临时抱佛脚呢，还是应该 move on？
